# 프로토타입 연결 — 디자이너용 치트시트

> 피그마 화면들을 **말로 이어 붙여** 살아있는 프로토타입으로 만드는 도구(figma-prototype-mcp) 사용법.
> 손으로 화살표 드래그 → 트리거 선택 → 애니메이션 선택… 반복하던 걸 한 문장으로.
>
> 기준: v0.26.0 · 갱신: 2026-06-11

---

## 한 줄 요약
**있는 화면을 "연결"하는 도구입니다.** 화면·버튼을 "만드는" 건 공식 Figma MCP가 맡고, 이 도구가 그걸 "잇습니다" — **둘을 같이 쓰는 걸 전제**로 한 역할 분담이에요. (아래 "함께 쓰기" 참고)

## ✨ 이렇게 말하면 이렇게 돼요

| 이렇게 말하면 | 이렇게 됩니다 |
|---|---|
| "screen1에서 screen2로 넘어가게" | 클릭하면 화면 전환 |
| "옆에서 밀고 들어오게 / 위로 올라오게" | 미는·올라오는 방향 애니메이션 |
| "부드럽게 / 통통 튀게 / 빠르게 / iOS 느낌으로" | 말한 느낌에 맞는 모션 자동 선택 |
| "메뉴 버튼 누르면 팝업 뜨고, 닫기도" | 오버레이(위에 뜨는 창) 열기+닫기 |
| "이 화면에 뒤로가기 달아줘" | 좌상단 백버튼을 **알아서 찾아** 연결 |
| "스와이프로 뒤로 가게" | 화면 밀어서 뒤로 (제스처) |
| "이 페이지 아래로 스크롤되게" | 스크롤 위치 이동 |
| "로그인하면 상태 true로, 홈에서 분기" | 변수(상태) 설정 + 조건 분기 |
| "여기 누르면 사이트 열리게" | 외부 링크 열기 |
| "방금 만든 거 좀 더 느리게" | 직전 연결을 찾아 수정 (※ 가끔 안 잡힘) |

**트리거**(언제): 클릭 / 스와이프 / 마우스 올렸을 때 / 일정 시간 후 등
**전환**(어떻게): 즉시 · 페이드 · 스마트애니메이트(모핑) · 방향전환(밀기/슬라이드) + 속도·방향

## 🧠 알아서 똑똑하게

- **모션 자동 추천** — 안 정하면 가장 자연스러운 전환을 골라줌.
- **안 어울리면 자동으로 바꿔줌** — 두 화면 구성이 너무 달라 모핑이 어색할 상황이면 **부드러운 페이드로 바꾸고 "왜 바꿨는지" 알려줌.** (모르고 깨진 모션 쓰는 걸 방지)
- **백버튼 스스로 찾기** — 좌상단 백 아이콘을 위치·모양으로 발견. 없으면 *"백버튼이 안 보이는데 스와이프로 할까요?"* 하고 **물어봄.**
- **애매하면 멋대로 안 하고 물어봄.**

## 🤝 공식 Figma MCP와 같이 쓰기 (역할 분담)

이 도구는 일부러 **"연결" 한 가지에 집중**합니다. 화면·버튼을 새로 **만드는 건 공식 Figma MCP**(`use_figma` 등)가 맡아요. 둘을 같은 자리에서 함께 쓰면 자연스럽게 분업됩니다:

- *"장바구니 화면 만들고 결제로 연결까지 해줘"* → 공식 MCP가 화면을 **만들고** → 이 도구가 **이어 붙임.**
- 그래서 "화면 생성"은 이 도구의 빈칸이 아니라 **설계상 공식 MCP의 몫**이에요. 두 도구를 한 환경에서 쓰는 걸 전제로 합니다.

참고 — 피그마 자체 규칙에서 오는 제약은 그대로 따라요 (예: 오버레이엔 스마트애니메이트가 원래 안 먹어서 자동 페이드 처리).

## 💡 잘 쓰는 팁

- 화면은 **이름으로 불러도** 됨 ("screen1 → screen2"). 알아서 찾아 연결.
- 더 정확히 하려면 연결할 **화면을 미리 선택**해두고 말하면 좋음.
- 한 번에 여러 개도 OK ("각 화면에 다 뒤로가기 달아줘").
- 결과가 마음에 안 들면 그냥 *"방금 거 스와이프로 바꿔줘"* 처럼 이어서 말하면 됨.
