# v1.10 AFTER_TIMEOUT trigger — Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development.

**Goal:** Add the AFTER_TIMEOUT trigger to `create_reactions` so reactions can fire automatically after a delay. Pairs naturally with every existing action variant (navigate, scroll, overlay, close, back, url, swap_overlay).

**Architecture:** Extract a shared `buildTrigger` helper. Widen the trigger enum + add `afterTimeoutSeconds` to ConnectionInput and every builder. Zod refine enforces the conditional requirement. `list_reactions` echoes the `timeout` value.

**Source spec:** `docs/superpowers/specs/2026-05-20-v1.10-after-timeout-trigger-design.md`

- [ ] **Step 3: Update `src/figma-plugin/reaction-builder.ts`**

Replace the existing `TriggerName` and add helper:

```typescript
export type TriggerName = "ON_CLICK" | "ON_HOVER" | "ON_PRESS" | "AFTER_TIMEOUT";

export type TriggerShape = { type: string; timeout?: number };

export function buildTrigger(name: TriggerName, afterTimeoutSeconds?: number): TriggerShape {
  if (name === "AFTER_TIMEOUT") {
    if (afterTimeoutSeconds === undefined) {
      throw new Error("buildTrigger: afterTimeoutSeconds is required when name is AFTER_TIMEOUT");
    }
    return { type: "AFTER_TIMEOUT", timeout: afterTimeoutSeconds };
  }
  return { type: name };
}
```

Update `BuiltReaction.trigger` type:

```typescript
export interface BuiltReaction {
  trigger: TriggerShape;
  actions: BuiltAction[];
}
```

Add `afterTimeoutSeconds?: number` to each BuildInput interface. Example for NavigateBuildInput:

```typescript
export interface NavigateBuildInput {
  targetFrameId: string;
  trigger: TriggerName;
  transition: TransitionName;
  afterTimeoutSeconds?: number;
}
```

Do the same for `ScrollBuildInput`, `OverlayBuildInput`, `CloseBuildInput`, `BackBuildInput`, `UrlBuildInput`, `SwapOverlayBuildInput` — all 7 interfaces get the same new field.

In each of the 7 builder functions, replace:

```typescript
trigger: { type: input.trigger },
```

with:

```typescript
trigger: buildTrigger(input.trigger, input.afterTimeoutSeconds),
```
