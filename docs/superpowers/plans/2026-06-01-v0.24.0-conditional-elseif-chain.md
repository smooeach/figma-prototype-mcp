# v0.24.0 — conditional `else-if` chain Implementation Plan

> ⚠️ **ABANDONED (2026-06-01) — blocked by Figma platform.** Tasks 1–5 were implemented and fully green (357 tests + typecheck), but Task 6 live verification proved Figma's prototype has no else-if (UI offers only If/Else; `setReactionsAsync` collapses a 3-block conditional to 2 — `__builtBlocks: 3` → `__storedBlocks: 2`). Implementation commits were reverted. This plan is kept as a historical record and a ready blueprint if Figma ever adds else-if. See the **Outcome** section of the design spec (`docs/superpowers/specs/2026-06-01-v0.24.0-conditional-elseif-chain-design.md`).

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the `conditional` action from a single if/else into an ordered **if / else-if* / else** chain, on both the low-level `create_reactions` conditional action and the high-level `proto_conditional` tool.

**Architecture:** Backward-compatible additive change. A new optional `elseIf` array is inserted between `then` and `else` in the low-level zod schema; the plugin builder (already block-array based) pushes each elseIf block in order before the condition-less else block; the list_reactions decoder is generalized from the 2-block cap to N blocks; `proto_conditional` gains an `elseIf` sugar that compiles to the low-level shape. Each block stays a single comparison (no compound conditions). Nesting and `toggle_variable`-inside-conditional remain rejected.

**Tech Stack:** TypeScript + Zod + Vitest + @modelcontextprotocol/sdk. No new dependencies.

**Source spec:** `docs/superpowers/specs/2026-06-01-v0.24.0-conditional-elseif-chain-design.md`

**Designer milestone protocol:** Each task ends with a one-line "디자이너 요약" note. Implementation subagents must include this note in their report.

```ts
export interface ConditionalBuildInput {
  trigger: TriggerInput;
  afterTimeoutSeconds?: number;
  condition: unknown;             // VariableData; constructed by the plugin
  thenActions: BuiltAction[];
  elseIfBlocks?: { condition: unknown; actions: BuiltAction[] }[];
  elseActions?: BuiltAction[];
}

export function buildConditionalReaction(input: ConditionalBuildInput): BuiltReaction {
  const blocks: ConditionalBlockShape[] = [
    { condition: input.condition, actions: input.thenActions },
  ];
  for (const b of input.elseIfBlocks ?? []) {
    blocks.push({ condition: b.condition, actions: b.actions });
  }
  if (input.elseActions && input.elseActions.length > 0) {
    blocks.push({ actions: input.elseActions });
  }
  return {
    trigger: buildTrigger(input.trigger, input.afterTimeoutSeconds),
    actions: [{ type: "CONDITIONAL", conditionalBlocks: blocks }],
  };
}
```

## Task 4: Plugin decoder — generalize `encodeActionForListEcho` to N blocks

**Files:**

- [ ] **Step 1: Replace the standardPattern decoder**

In `encodeActionForListEcho`, the conditional branch currently has (after the `detectTogglePattern` early-return) a block beginning `// 2) Fall back to existing conditional decoder` and ending with the `return { type: "CONDITIONAL", condition, then, else }`. Replace that entire block (currently lines 783-806) with:

```ts
    // 2) Fall back to conditional decoder — generalized to N blocks (if / else-if* / else).
    // Standard shape: block[0] has a condition; only the LAST block may be condition-less (the else).
    const lastIdx = blocks.length - 1;
    const standardPattern =
      blocks.length >= 1 &&
      blocks[0].condition !== undefined &&
      blocks.every((b: any, i: number) => (i === lastIdx ? true : b.condition !== undefined));

    if (!standardPattern) {
      // Non-standard shape (e.g. a condition-less block in the middle) — return raw for inspection.
      return { type: "CONDITIONAL", raw: blocks };
    }

    const decodedCondition = await decodeConditionForEcho(blocks[0].condition);
    const thenActions = await Promise.all(
      (blocks[0].actions ?? []).map((a: any) => encodeActionForListEcho(a))
    );

    // A trailing condition-less block (when there are ≥2 blocks) is the else.
    const hasElse = blocks.length >= 2 && blocks[lastIdx].condition === undefined;
    const elseActions = hasElse
      ? await Promise.all((blocks[lastIdx].actions ?? []).map((a: any) => encodeActionForListEcho(a)))
      : undefined;

    // Intermediate condition-bearing blocks (index 1 .. before a trailing else) are else-if.
    const elseIfEnd = hasElse ? lastIdx : blocks.length;
    const elseIfDecoded: { condition: unknown; then: unknown[] }[] = [];
    for (let i = 1; i < elseIfEnd; i++) {
      elseIfDecoded.push({
        condition: await decodeConditionForEcho(blocks[i].condition),
        then: await Promise.all((blocks[i].actions ?? []).map((a: any) => encodeActionForListEcho(a))),
      });
    }

    return {
      type: "CONDITIONAL",
      condition: decodedCondition,
      then: thenActions,
      ...(elseIfDecoded.length > 0 && { elseIf: elseIfDecoded }),
      else: elseActions,
    };
```

Note: for a v0.23.0-style 2-block reaction (then + else), `elseIfDecoded` is empty → `elseIf` key omitted, and the echo is `{ type, condition, then, else }` — byte-identical to before. For a 1-block reaction, `else` is `undefined` → omitted by JSON. This preserves the existing round-trip exactly.

```ts
export function compileProtoConditional(input: ProtoConditionalInput): CreateReactionsInputType {
  const connections: Connection[] = input.conditions.map((c) => {
    const baseTransition = resolveMotion(c.motion);
    const needsOverlayRewrite =
      branchUsesOverlayOrSwap(c.then) ||
      (c.elseIf ?? []).some((ei) => branchUsesOverlayOrSwap(ei.then)) ||
      (c.else !== undefined && branchUsesOverlayOrSwap(c.else));
    const transition = needsOverlayRewrite ? rewriteForOverlay(baseTransition) : baseTransition;

    const action: Connection["action"] = {
      type: "conditional",
      condition: {
        variable: c.if.variable,
        operator: c.if.operator,        // zod already applied default "=="
        value: c.if.value,
      },
      then: [compileBranchAction(c.then)],
      ...(c.elseIf !== undefined && {
        elseIf: c.elseIf.map((ei) => ({
          condition: { variable: ei.if.variable, operator: ei.if.operator, value: ei.if.value },
          then: [compileBranchAction(ei.then)],
        })),
      }),
      ...(c.else !== undefined && { else: [compileBranchAction(c.else)] }),
    };

    return {
      sourceNodeId: c.from,
      trigger: c.trigger ?? DEFAULT_TRIGGER,
      transition,
      action,
    } as Connection;
  });
  return { connections, replaceExisting: input.replaceExisting };
}
```
