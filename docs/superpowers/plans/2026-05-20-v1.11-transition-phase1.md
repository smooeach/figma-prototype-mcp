# v1.11 Transition Phase 1 — Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development.

**Goal:** Replace fixed `0.3s + EASE_OUT` with user-controllable duration and 7 cubic/back easings while keeping `"INSTANT"` / `"DISSOLVE"` / `"SMART_ANIMATE"` shortcuts working unchanged. Simple transitions only — directional + spring + custom land in later phases.

**Architecture:** `buildTransition` is rewritten to accept a union (shortcut string or nested object). The 4 transition-aware builders (`buildNavigateReaction`, `buildScrollReaction`, `buildOverlayReaction`, `buildSwapOverlayReaction`) pass the transition input through unchanged — only the builder's internal resolution changes. Zod `TransitionInput` is `z.union([z.literal(...), z.literal(...), z.literal(...), SimpleTransitionObject])`. Easing emitted as a nested `{ type: ... }` object per Figma's wire shape.

**Source spec:** `docs/superpowers/specs/2026-05-20-v1.11-transition-phase1-design.md`

Replace `TransitionName` and `TransitionShape` blocks with:

```typescript
export type TransitionName = "INSTANT" | "DISSOLVE" | "SMART_ANIMATE";
// shortcut strings — keep for backward compat at the API surface

export type SimpleTransitionType = "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE";

export type EasingName =
  | "LINEAR" | "EASE_IN" | "EASE_OUT" | "EASE_IN_AND_OUT"
  | "EASE_IN_BACK" | "EASE_OUT_BACK" | "EASE_IN_AND_OUT_BACK";

export interface SimpleTransitionInput {
  type: SimpleTransitionType;
  duration?: number;       // seconds, default 0.3
  easing?: EasingName;     // default "EASE_OUT"
}

export type TransitionInput = TransitionName | SimpleTransitionInput;

export type TransitionShape =
  | null
  | {
      type: SimpleTransitionType;
      duration: number;
      easing: { type: EasingName };
    };
```

Replace the `buildTransition` function:

```typescript
export function buildTransition(input: TransitionInput): TransitionShape {
  if (input === "INSTANT") return null;

  // Shortcut strings expand to nested with defaults.
  if (input === "DISSOLVE" || input === "SMART_ANIMATE") {
    return {
      type: input,
      duration: 0.3,
      easing: { type: "EASE_OUT" },
    };
  }

  // Nested object — resolve defaults.
  const duration = input.duration ?? 0.3;
  const easing: EasingName = input.easing ?? "EASE_OUT";
  return {
    type: input.type,
    duration,
    easing: { type: easing },
  };
}
```

Update the 4 transition-accepting BuildInput interfaces to use `TransitionInput` instead of `TransitionName`:
- `NavigateBuildInput`
- `ScrollBuildInput`
- `OverlayBuildInput`
- `SwapOverlayBuildInput`

Example:
```typescript
export interface NavigateBuildInput {
  targetFrameId: string;
  trigger: TriggerName;
  transition: TransitionInput;     // ← was TransitionName
  afterTimeoutSeconds?: number;
}
```

Do the same for the other 3 interfaces. `CloseBuildInput`, `BackBuildInput`, `UrlBuildInput` do not have transition — leave them.

In the 4 affected builder functions, the existing `buildTransition(input.transition)` call now accepts the wider union — no change needed at the call site.
