# v1.8 Plugin auto-reconnect — Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development.

**Goal:** Add automatic WebSocket reconnect with exponential backoff to the plugin UI. Distinguishes user-triggered Disconnect (no retry) from unexpected close (retry until success or user intervention).

**Architecture:** Single-file change in `src/figma-plugin/ui.html`. State machine driven by `userDisconnected` flag + `retryTimer` + `retryDelay` variables.

**Source spec:** `docs/superpowers/specs/2026-05-20-v1.8-auto-reconnect-design.md`

## Task 1: UI state + retry loop

**Files:**
- Modify: `src/figma-plugin/ui.html`

- [ ] **Step 1: Add retry state variables**

Inside the `<script>` block, near the top (right after `let ws = null;`), add:

```javascript
      let userDisconnected = false;
      let retryTimer = null;
      let retryDelay = 1000;
      const MAX_RETRY_DELAY = 30000;
```

- [ ] **Step 2: Update `connect()` to reset retry state on entry**

The current `connect()` starts:

```javascript
      function connect() {
        const channel = channelInput.value.trim();
        if (!channel) { setStatus('Channel name required', 'err'); return; }
        if (ws) ws.close();
        setStatus('Connecting…', 'pending');
        ws = new WebSocket('ws://localhost:3055');
        ...
      }
```

Add two lines at the very top of `connect()`:

```javascript
      function connect() {
        if (retryTimer) { clearTimeout(retryTimer); retryTimer = null; }
        userDisconnected = false;
        const channel = channelInput.value.trim();
        ...
```

- [ ] **Step 3: Rewrite `ws.onclose` to schedule a retry**

The current `ws.onclose` reads:

```javascript
        ws.onclose = () => {
          if (statusEl.className !== 'err') setStatus('Disconnected', 'err');
          setDisconnectedUI();
        };
```

Replace with:

```javascript
        ws.onclose = () => {
          if (userDisconnected) {
            setStatus('Disconnected', 'err');
            setDisconnectedUI();
            return;
          }
          const channel = channelInput.value.trim();
          setStatus('Reconnecting to ' + channel + ' in ' + (retryDelay / 1000) + 's…', 'pending');
          retryTimer = setTimeout(() => {
            retryTimer = null;
            setStatus('Reconnecting to ' + channel + '…', 'pending');
            retryDelay = Math.min(retryDelay * 2, MAX_RETRY_DELAY);
            connect();
          }, retryDelay);
        };
```

(Note: input stays disabled and Disconnect button stays visible during the retry loop — we do NOT call `setDisconnectedUI()` when scheduling a retry. The UI looks like it's still in the "connected" shape, just with a "reconnecting…" status.)

- [ ] **Step 4: Reset backoff in `setConnectedUI`**

The current `setConnectedUI` reads:

```javascript
      function setConnectedUI(channel) {
        setStatus('Connected on channel: ' + channel, 'ok');
        channelInput.disabled = true;
        connectBtn.style.display = 'none';
        disconnectBtn.style.display = 'inline-block';
      }
```

Append one line:

```javascript
      function setConnectedUI(channel) {
        setStatus('Connected on channel: ' + channel, 'ok');
        channelInput.disabled = true;
        connectBtn.style.display = 'none';
        disconnectBtn.style.display = 'inline-block';
        retryDelay = 1000;
      }
```

- [ ] **Step 5: Update `disconnectBtn.onclick`**

The current handler reads:

```javascript
      disconnectBtn.onclick = () => {
        if (ws) ws.close();
        setStatus('Disconnected', 'err');
        setDisconnectedUI();
      };
```

Replace with:

```javascript
      disconnectBtn.onclick = () => {
        userDisconnected = true;
        if (retryTimer) { clearTimeout(retryTimer); retryTimer = null; }
        if (ws) ws.close();
        setStatus('Disconnected', 'err');
        setDisconnectedUI();
      };
```

(`userDisconnected = true` is set BEFORE `ws.close()` so the resulting `onclose` event takes the user-disconnect branch.)

```bash
npm run build:plugin
npm test
npx tsc --noEmit
```

Expected: build success, 53 tests pass, tsc clean.

feat(plugin-ui): auto-reconnect with exponential backoff on unintended close

Unexpected ws.onclose (any close that isn't from the Disconnect

disabled, Disconnect visible) so the user can cancel by clicking
