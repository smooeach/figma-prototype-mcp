# figma-prototype-mcp v1.12 — Spring presets + Custom easings 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.10.0
- 시리즈: 4-phase 트랜지션 노출 중 **Phase 2 + Phase 4 묶음** (Phase 3 directional은 별도)
- 참조: `docs/natural-language-mapping-dictionary-v2.3.md`, `docs/superpowers/specs/2026-05-20-v1.11-transition-phase1-design.md`

## 1. 목적

v0.10.0의 EasingInput은 7 cubic/back easings만 노출. Figma는 추가로 4 spring presets (GENTLE/QUICK/BOUNCY/SLOW) + 2 custom shapes (CUSTOM_CUBIC_BEZIER / CUSTOM_SPRING) 지원. v1.12는 이 둘을 한 번에 추가해서 사용자가 자연어로 "playful bounce" / "Material 3 emphasized" / "very springy" 같은 표현을 정확한 Figma 값으로 매핑 가능하게 함.

Phase 3 directional은 새 transition shape 자체가 다르므로 별도 release.

## 2. spec-단계 readonly/setter 체크

Easing 객체는 transition 안에 nested 되어 setReactionsAsync wire에 들어감. v1.11 패턴과 동일. ✅ 차단 없음.

추가 검증: Figma typings 5193–5200에서

```ts
interface Easing {
  type: ...
  easingFunctionCubicBezier?: { x1, y1, x2, y2 }
  easingFunctionSpring?: { mass, stiffness, damping, initialVelocity }
}
```

— 두 nested field가 정확히 정의되어 있어 라이브 거부 위험 낮음.

## 3. 스코프

### v1.12 포함
- **4 spring presets**: `"GENTLE"`, `"QUICK"`, `"BOUNCY"`, `"SLOW"` — easing input 문자열 enum에 추가, builder는 `{ type }`만 emit
- **CUSTOM_CUBIC_BEZIER**: 사용자 input은 flat `{ type, x1, y1, x2, y2 }`, builder는 `{ type, easingFunctionCubicBezier: { x1, y1, x2, y2 } }`로 nesting 추가
- **CUSTOM_SPRING**: 사용자 input은 flat `{ type, mass, stiffness, damping, initialVelocity }`, builder는 `{ type, easingFunctionSpring: { ... } }`로 nesting
- Validation:
  - cubic-bezier x1/x2: `z.number().min(0).max(1)` (Zod refine 아닌 chain)
  - cubic-bezier y1/y2: `z.number()` (back/overshoot 허용)
  - spring mass/stiffness/damping: `z.number().positive()`
  - spring initialVelocity: `z.number()` (음수 허용 — 반대 방향 초기속도)
- `list_reactions` 응답의 easing 객체는 raw passthrough (Figma가 돌려주는 nested shape 그대로)
- 새 E2E 시나리오 15 (Spring) + 16 (Custom)

### v1.12 비-목표
- **Phase 3 directional transitions** (MOVE_IN/OUT, PUSH, SLIDE_IN/OUT, direction, matchLayers) — 별도 release
- Spring preset과 cubic-bezier을 자동 비교/추천 (e.g., "이 BOUNCY가 stiffness 400 spring과 비슷" 같은 안내)

## 4. 사용자 API

### 4.1 EasingInput 전체 union

```ts
type EasingInput =
  // 7 cubic/back (v0.10.0)
  | "LINEAR" | "EASE_IN" | "EASE_OUT" | "EASE_IN_AND_OUT"
  | "EASE_IN_BACK" | "EASE_OUT_BACK" | "EASE_IN_AND_OUT_BACK"
  // 4 spring presets (NEW)
  | "GENTLE" | "QUICK" | "BOUNCY" | "SLOW"
  // 2 custom (NEW) — flat user input
  | { type: "CUSTOM_CUBIC_BEZIER", x1: number, y1: number, x2: number, y2: number }
  | { type: "CUSTOM_SPRING", mass: number, stiffness: number, damping: number, initialVelocity: number };
```

### 4.2 Wire shape 매핑

| EasingInput | Wire shape (Figma 받는 형식) |
|---|---|
| `"GENTLE"` / `"QUICK"` / `"BOUNCY"` / `"SLOW"` | `{ type: "<NAME>" }` (field 추가 없음) |
| `{ type: "CUSTOM_CUBIC_BEZIER", x1, y1, x2, y2 }` | `{ type: "CUSTOM_CUBIC_BEZIER", easingFunctionCubicBezier: { x1, y1, x2, y2 } }` |
| `{ type: "CUSTOM_SPRING", mass, stiffness, damping, initialVelocity }` | `{ type: "CUSTOM_SPRING", easingFunctionSpring: { mass, stiffness, damping, initialVelocity } }` |

### 4.3 사용 예시

```ts
// 1) Spring preset
transition: { type: "SMART_ANIMATE", duration: 0.4, easing: "BOUNCY" }
// 2) Material 3 emphasized cubic-bezier
transition: { type: "SMART_ANIMATE", duration: 0.5, easing: { type: "CUSTOM_CUBIC_BEZIER", x1: 0.2, y1: 0, x2: 0, y2: 1 } }
// 3) Very-bouncy custom spring
transition: { type: "SMART_ANIMATE", duration: 0.6, easing: { type: "CUSTOM_SPRING", mass: 1, stiffness: 600, damping: 10, initialVelocity: 0 } }
```

## 5. Zod 스키마

```ts
const NamedEasingEnum = z.enum([
  "LINEAR", "EASE_IN", "EASE_OUT", "EASE_IN_AND_OUT",
  "EASE_IN_BACK", "EASE_OUT_BACK", "EASE_IN_AND_OUT_BACK",
  "GENTLE", "QUICK", "BOUNCY", "SLOW",
]);

const CustomCubicBezierEasing = z.object({
  type: z.literal("CUSTOM_CUBIC_BEZIER"),
  x1: z.number().min(0).max(1),
  y1: z.number(),
  x2: z.number().min(0).max(1),
  y2: z.number(),
});

const CustomSpringEasing = z.object({
  type: z.literal("CUSTOM_SPRING"),
  mass: z.number().positive(),
  stiffness: z.number().positive(),
  damping: z.number().positive(),
  initialVelocity: z.number(),
});

const EasingInputUnion = z.union([NamedEasingEnum, CustomCubicBezierEasing, CustomSpringEasing]);

// SimpleTransitionObject.easing은 이제 EasingInputUnion.optional()
```

## 6. 빌더 함수 변화

`buildTransition`이 easing을 wire shape으로 변환할 때:

```ts
function resolveEasing(easing: EasingInput | undefined): EasingShape {
  if (easing === undefined) return { type: "EASE_OUT" };
  if (typeof easing === "string") return { type: easing };
  // Custom object — flat input, wrap with nested field name
  if (easing.type === "CUSTOM_CUBIC_BEZIER") {
    return {
      type: "CUSTOM_CUBIC_BEZIER",
      easingFunctionCubicBezier: { x1: easing.x1, y1: easing.y1, x2: easing.x2, y2: easing.y2 },
    };
  }
  // CUSTOM_SPRING
  return {
    type: "CUSTOM_SPRING",
    easingFunctionSpring: {
      mass: easing.mass, stiffness: easing.stiffness,
      damping: easing.damping, initialVelocity: easing.initialVelocity,
    },
  };
}
```

기존 `buildTransition` 안에서 `easing: { type: easing }` 부분을 `resolveEasing(input.easing)`으로 교체.

`EasingShape` 타입 확장:

```ts
type EasingShape =
  | { type: NamedEasingName }
  | { type: "CUSTOM_CUBIC_BEZIER", easingFunctionCubicBezier: { x1: number, y1: number, x2: number, y2: number } }
  | { type: "CUSTOM_SPRING", easingFunctionSpring: { mass: number, stiffness: number, damping: number, initialVelocity: number } };

type NamedEasingName =
  | "LINEAR" | "EASE_IN" | "EASE_OUT" | "EASE_IN_AND_OUT"
  | "EASE_IN_BACK" | "EASE_OUT_BACK" | "EASE_IN_AND_OUT_BACK"
  | "GENTLE" | "QUICK" | "BOUNCY" | "SLOW";
```

`TransitionShape.easing`이 위 union을 갖도록 widen.

## 7. Plugin handler 변화

`Command` union의 transition 필드 안 easing 타입 확장 — string union 11개 + 2개 object variants.

`handleListReactions`의 transition echo는 raw passthrough이므로 변경 없음 (Figma가 돌려주는 easing 객체에 `easingFunctionCubicBezier`/`easingFunctionSpring` 포함된 채로 전달됨).
