# figma-prototype-mcp v1.7 — Plugin auto-connect 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.5.0
- 영향 범위: figma-plugin UI/sandbox만. MCP server / relay / 빌더 / 핸들러 무관.

## 1. 목적

매 세션마다 Figma plugin을 띄울 때 채널명을 다시 타이핑하고 Connect 버튼을 눌러야 함 — 가장 잦은 마찰. `figma.clientStorage`로 마지막 채널을 영속화하고 plugin 실행 시 자동 connect. v2 plugin-ization과 직교 (v2에서 갈아엎혀도 학습 손실 없음).

## 2. spec-단계 readonly/setter 체크

```ts
// @figma/plugin-typings (line 2568-2585)
interface ClientStorageAPI {
  getAsync(key: string): Promise<any | undefined>
  setAsync(key: string, value: any): Promise<void>
  deleteAsync(key: string): Promise<void>
  keysAsync(): Promise<string[]>
}
```

✅ `setAsync` / `deleteAsync` 정상 mutation API. v1.4 readonly 차단 없음.

## 3. 스코프

### v1.7 포함

- Plugin UI에서 sandbox로 `load-channel` 메시지 → sandbox가 clientStorage에서 채널 읽어 응답
- Sandbox에서 `save-channel` 메시지 받으면 clientStorage에 저장
- UI 로드 시 자동으로 `load-channel` 요청 → 응답에 채널 있으면 input 채우고 **자동 Connect 시작**
- Join 성공 시 자동으로 `save-channel` 발송
- UI에 **Disconnect** 버튼 추가 (WS 닫고 input 재편집 가능)

### v1.7 비-목표

- **Auto-reconnect** (relay 끊겼다 살아나면 자동 재연결) — v1.8 후보
- 다중 채널 history / dropdown
- 채널 forget 옵션 (clientStorage.deleteAsync 노출)
- Figma plugin parameters (quick action 직접 실행) — 별도

## 4. 메시지 프로토콜

기존 sandbox ↔ UI 메시지에 두 종류 추가. 모두 `figma.ui.postMessage` / `parent.postMessage({pluginMessage: ...})`로 주고받음.

### 4.1 UI → sandbox

| type | 페이로드 | 의미 |
|---|---|---|
| `load-channel` | (없음) | 저장된 채널을 읽어달라 |
| `save-channel` | `{ channel: string }` | 이 채널을 clientStorage에 저장 |
| `command` (기존) | `{ envelope: ... }` | 기존 명령 forwarding |

### 4.2 sandbox → UI

| type | 페이로드 | 의미 |
|---|---|---|
| `channel-loaded` | `{ channel: string \| null }` | clientStorage 읽기 결과. null이면 저장된 값 없음 |
| `response` (기존) | `{ envelope: ... }` | 기존 명령 응답 |

## 5. UI 상태 흐름

```
[idle, input disabled]
  ↓ DOM ready
  → send "load-channel"
  ↓ receive "channel-loaded"
   ├─ channel != null:
   │    input.value = channel
   │    input.disabled = true
   │    auto-trigger connect()
   └─ channel == null:
        input.disabled = false   (user-typeable, ready for manual Connect)

[connecting]
  ↓ WS open + join ACK 수신
  → send "save-channel" with current input value
  → status = "Connected on channel: X"
  → show Disconnect button, hide Connect button
  → input.disabled = true

[connected]
  ↓ Disconnect 클릭
  → ws.close()
  → input.disabled = false
  → show Connect button, hide Disconnect
  → status = "Disconnected" (또는 빈 상태)
  → (clientStorage는 그대로 둠 — 다음 launch는 여전히 auto-connect)
```

연결 실패 처리:
- WS open 에러 → 에러 메시지 노출, input 다시 편집 가능, Disconnect 버튼 없음
- 저장된 채널 있는 상태에서 auto-connect 실패해도 clientStorage는 그대로 둠 (사용자가 명시적으로 forget 안 함)

## 6. Sandbox 코드 변경

`src/figma-plugin/code.ts`의 `figma.ui.onmessage` 핸들러는 현재 `command` 타입만 처리. 두 케이스 추가:

```ts
figma.ui.onmessage = (msg: any) => {
  if (msg?.type === "load-channel") {
    figma.clientStorage.getAsync("channel").then((value) => {
      figma.ui.postMessage({
        type: "channel-loaded",
        channel: typeof value === "string" ? value : null,
      });
    });
    return;
  }
  if (msg?.type === "save-channel" && typeof msg.channel === "string") {
    figma.clientStorage.setAsync("channel", msg.channel).catch(() => { /* swallow */ });
    return;
  }
  if (msg?.type === "command" && msg.envelope) {
    // 기존 command 처리 (변경 없음)
    ...
  }
};
```

clientStorage 호출은 sandbox에서만 가능 (UI iframe 사이드에서는 안 됨) — postMessage 우회가 필수.

## 7. UI 코드 변경

`src/figma-plugin/ui.html`:

- DOM ready 시 `parent.postMessage({ pluginMessage: { type: 'load-channel' } }, '*')`
- `window.onmessage`에 `channel-loaded` 케이스 추가 — pre-fill + auto-connect
- 기존 `connect` 클릭 핸들러는 유지하되, 함수로 추출 (`connect()`)해 auto-connect와 공유
- 기존 `set-channel` 메시지를 `save-channel`로 rename. join ACK 시 발송 (현재 코드의 `set-channel`은 의도는 같지만 sandbox에서 무시되었음 — 이번에 활용)
- Disconnect 버튼 신설. 초기에는 hidden, 연결 후 보임.
