# v0.21.0 — `proto_back` + `proto_url`

Status: design (2026-05-22).
Predecessor: v0.20.1 (polish bundle).
Release type: minor.
Scope: add two atomic proto_* tools that complete the high-level navigation cluster (wire / overlay / scroll → + back + url). No new low-level actions — both wrap existing action types from v1.5.

## Goal

Lift the existing `back` and `url` action types from the `create_reactions` escape hatch into the `proto_*` namespace as two intent-oriented tools. Use cases like "make this button go back" or "open this link" no longer require the LLM to construct a low-level `create_reactions` payload — a one-liner `proto_back` or `proto_url` call suffices.

## Decisions (from brainstorm)

- **Tool count**: 2 atomic tools (`proto_back` + `proto_url`), not consolidated. Each action type is conceptually independent — no shared "navigate-like" abstraction.
- **`proto_url` has NO `motion` field**: opening a URL is a terminal event in Figma's prototype runtime. The reaction's transition field is technically present but visually inert. Excluding `motion` keeps the schema honest and avoids implying the user can affect the transition.
- **Both tools record to HistoryStore**: proto_back and proto_url join the recording cohort, joining proto_wire / proto_overlay / proto_scroll. `ProtoToolName` grows from 3 entries to 5.
- **Korean disclaimer prefix**: both descriptions start with the same usage-restriction line as the other proto_* tools (established in v1.21).

## Section 2 — Tool signatures

### `proto_back`

```ts
proto_back({
  backs: [
    {
      from: string,                  // sourceNodeId
      trigger?: TriggerInput,        // default "ON_CLICK"
      motion?: MotionInput,          // default "M3_EMPHASIZED"
    }
  ],
  replaceExisting?: boolean,
})
```

Each entry compiles to a `create_reactions` connection with:
```ts
{
  sourceNodeId: from,
  trigger: trigger ?? "ON_CLICK",
  transition: resolveMotion(motion),
  action: { type: "back" },
}
```

Same motion preset machinery as proto_wire. The Figma BACK action pops the prototype history stack — no destination is needed (the runtime knows where "back" points).

### `proto_url`

```ts
proto_url({
  urls: [
    {
      from: string,                  // sourceNodeId
      url: string,                   // must be a non-empty string (zod min(1))
      openInNewTab?: boolean,        // default false (matches existing UrlActionInput)
      trigger?: TriggerInput,        // default "ON_CLICK"
    }
  ],
  replaceExisting?: boolean,
})
```

Each entry compiles to a connection that **omits** `transition` (lets `create_reactions` zod's `.default("INSTANT")` take effect):
```ts
{
  sourceNodeId: from,
  trigger: trigger ?? "ON_CLICK",
  // transition deliberately not set; defaults to "INSTANT" via create_reactions schema
  action: { type: "url", url, openInNewTab: openInNewTab ?? false },
}
```

No `motion` field on input. URLs open externally — transition is visually inert. Documenting and enforcing this in the schema keeps the surface honest.

`openInNewTab` matches the existing low-level `url` action behavior. Defaulting at the compile layer (explicit `?? false`) rather than relying on the inner schema's default — the inner UrlActionInput marks `openInNewTab` as optional, so for clarity we set it explicitly.

## Section 3 — `ProtoToolName` extension

`src/server/history.ts` currently exports:
```ts
export type ProtoToolName = "proto_wire" | "proto_overlay" | "proto_scroll";
```

Extend to:
```ts
export type ProtoToolName = "proto_wire" | "proto_overlay" | "proto_scroll" | "proto_back" | "proto_url";
```

No other change to `HistoryStore` — the FIFO ring buffer behavior, `record()` semantics, and `getLast()` shape are all type-agnostic over `ProtoToolName`. Existing tests continue to pass.

## Section 4 — Handler registration in `src/server/tools.ts`

Inside `makeTools(historyStore)`, append 2 entries after the existing `proto_scroll` entry (so the proto_get_last_history entry stays last):

```ts
{
  name: "proto_back",
  description:
    "이 도구는 오직 피그마의 프로토타입 인터랙션/애니메이션 생성·수정 목적으로만 사용합니다. " +
    "Wire source nodes to the Back navigation action (pops the prototype history stack — no destination). " +
    "Defaults: trigger=ON_CLICK, motion=M3_EMPHASIZED. Compiles to create_reactions internally.",
  schema: ProtoBackInput,
  handler: async (input, session) => {
    const parsedInput = input as ProtoBackInput;
    return recordedHandler(historyStore, "proto_back", parsedInput, () =>
      session.sendCommand("CREATE_REACTIONS" as CommandName, compileProtoBack(parsedInput)),
    );
  },
},
{
  name: "proto_url",
  description:
    "이 도구는 오직 피그마의 프로토타입 인터랙션/애니메이션 생성·수정 목적으로만 사용합니다. " +
    "Wire source nodes to the Open URL action. Input `{ urls: [{ from, url, openInNewTab? }] }`. " +
    "Defaults: trigger=ON_CLICK, openInNewTab=false. No `motion` field — URL is a terminal " +
    "event and the underlying reaction's transition defaults to INSTANT. Compiles to create_reactions internally.",
  schema: ProtoUrlInput,
  handler: async (input, session) => {
    const parsedInput = input as ProtoUrlInput;
    return recordedHandler(historyStore, "proto_url", parsedInput, () =>
      session.sendCommand("CREATE_REACTIONS" as CommandName, compileProtoUrl(parsedInput)),
    );
  },
},
```

Both use the v0.20.1 `recordedHandler` helper. Both close over `historyStore` via the `makeTools` parameter.

The new tool ordering in TOOLS: `…, proto_wire, proto_overlay, proto_scroll, proto_back, proto_url, proto_get_last_history`. proto_get_last_history stays last so the read-only history tool sits separately from the recording cluster.

## Section 5 — Testing strategy

### Unit tests on top of the existing 246

**`tests/protoTools.test.ts`** — schema tests for the 2 new inputs.
- `ProtoBackInput`: accepts minimal `{ backs: [{ from }] }`; accepts `trigger` + `motion` overrides; rejects empty `backs` array; rejects missing `from`.
- `ProtoUrlInput`: accepts minimal `{ urls: [{ from, url }] }`; accepts `openInNewTab` + `trigger` overrides; rejects empty `urls` array; rejects missing `url`; rejects `url: ""` (zod min(1)); **rejects unknown `motion` field** (`.strict()` to enforce no-motion contract).

**`tests/protoTools-compile.test.ts`** — compile output snapshots.
- `compileProtoBack` minimal → `{ sourceNodeId, trigger: "ON_CLICK", transition: M3_EMPHASIZED, action: { type: "back" }}`. Round-trip through `CreateReactionsInput.safeParse`.
- `compileProtoBack` with HIG_SNAPPY → named spring easing on back navigation.
- `compileProtoUrl` minimal → connection WITHOUT `transition` field. Round-trip through `CreateReactionsInput.safeParse` — should pass because `transition` has a zod default.
- `compileProtoUrl` with `openInNewTab: true` → flag propagated to action.
- `compileProtoUrl` with `openInNewTab` omitted → action carries `openInNewTab: false`.

**`tests/protoTools-history.test.ts`** — extend the integration tests to cover proto_back / proto_url recording.
- proto_back success → `entries[-1].tool === "proto_back"`.
- proto_url success → `entries[-1].tool === "proto_url"`.

Expected total: 246 → ~265 (+19). Numbers approximate; the actual test count depends on how granularly the schema tests get split.
