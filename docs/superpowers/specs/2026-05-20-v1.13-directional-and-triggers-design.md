# figma-prototype-mcp v1.13 — Directional transitions (Phase 3) + 모든 새 Trigger 타입 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.11.0
- 시리즈: 트랜지션 4-phase 노출 중 **Phase 3** + 트리거 확장 동시 번들
- 참조: `node_modules/@figma/plugin-typings/plugin-api.d.ts` Transition/Trigger (line 5183–5224), `docs/natural-language-mapping-dictionary-v2.3.md`, 이전 spec v1.10 (AFTER_TIMEOUT), v1.11/v1.12 (transition+easing)

## 1. 목적

v0.11.0 기준 노출되는 transition은 SimpleTransition (`INSTANT`/`DISSOLVE`/`SMART_ANIMATE`/`SCROLL_ANIMATE`)뿐이고 trigger도 `ON_CLICK`/`ON_HOVER`/`ON_PRESS`/`AFTER_TIMEOUT` 4종만 노출. Figma는 추가로:

- **DirectionalTransition** 5종 × 4방향 × matchLayers (`MOVE_IN`, `MOVE_OUT`, `PUSH`, `SLIDE_IN`, `SLIDE_OUT` × `LEFT`/`RIGHT`/`TOP`/`BOTTOM`)
- **새 Trigger 7종**: `ON_DRAG`, `MOUSE_UP`, `MOUSE_DOWN`, `MOUSE_ENTER`, `MOUSE_LEAVE`, `ON_KEY_DOWN`, `ON_MEDIA_HIT`, `ON_MEDIA_END`

를 지원. v1.13은 두 영역을 한 release에 묶어서 노출. 자연어 표현 "오른쪽에서 슬라이드", "드래그하면", "마우스 호버 0.2초 뒤", "스페이스바 누르면", "비디오 끝나면" 등이 정확한 Figma 값으로 매핑되게 함.

## 2. spec-단계 readonly/setter 체크

Trigger/Transition 모두 reaction 안에 nested 되어 `setReactionsAsync` wire로 들어감 — frame readonly 이슈 영역 아님. v1.11/v1.12 패턴과 동일. ✅ 차단 없음.

Figma typings 인용 (plugin-api.d.ts 5185–5224):

```ts
interface DirectionalTransition {
  readonly type: 'MOVE_IN' | 'MOVE_OUT' | 'PUSH' | 'SLIDE_IN' | 'SLIDE_OUT'
  readonly direction: 'LEFT' | 'RIGHT' | 'TOP' | 'BOTTOM'
  readonly matchLayers: boolean
  readonly easing: Easing
  readonly duration: number
}
type Trigger =
  | { readonly type: 'ON_CLICK' | 'ON_HOVER' | 'ON_PRESS' | 'ON_DRAG' }
  | { readonly type: 'AFTER_TIMEOUT'; readonly timeout: number }
  | { readonly type: 'MOUSE_UP' | 'MOUSE_DOWN'; readonly delay: number }
  | { readonly type: 'MOUSE_ENTER' | 'MOUSE_LEAVE'; readonly delay: number; readonly deprecatedVersion: boolean }
  | { readonly type: 'ON_KEY_DOWN'; readonly device: 'KEYBOARD'|'XBOX_ONE'|'PS4'|'SWITCH_PRO'|'UNKNOWN_CONTROLLER'; readonly keyCodes: ReadonlyArray<number> }
  | { readonly type: 'ON_MEDIA_HIT'; readonly mediaHitTime: number }
  | { readonly type: 'ON_MEDIA_END' }
```

추가 runtime 거부 위험:
- `matchLayers`, `deprecatedVersion`, `delay`는 typings에서 required지만 v1.12 `initialVelocity` 케이스처럼 runtime이 거부할 가능성. 라이브 검증 단계에서 1차 확인.
- `ON_KEY_DOWN`은 controller/key API라 prototype play에서 trigger 동작이 데스크탑 빌드 한정일 수 있음. spec에 검증 deferred 표시.
- `ON_MEDIA_HIT` / `ON_MEDIA_END`는 비디오 노드가 있어야 trigger 의미 있음 → schema/builder는 추가하되 라이브 검증은 deferred.

## 3. 스코프

### v1.13 포함

**Directional transitions (Phase 3):**
- `MOVE_IN`, `MOVE_OUT`, `PUSH`, `SLIDE_IN`, `SLIDE_OUT` 5 types
- `direction: "LEFT" | "RIGHT" | "TOP" | "BOTTOM"` 4 directions
- `matchLayers?: boolean` default `false`
- `duration?: number` default `0.3`
- `easing?: EasingInput` default `"EASE_OUT"` (v1.12 EasingInputUnion 재사용)
- Backward compat 영향 없음 — 새 type 값들이라 union 확장만

**새 Trigger 7종:**
- `ON_DRAG` — 추가 필드 없음
- `MOUSE_UP`, `MOUSE_DOWN` — `delay?: number` default `0`
- `MOUSE_ENTER`, `MOUSE_LEAVE` — `delay?: number` default `0`, `deprecatedVersion?: boolean` default `false`
- `ON_KEY_DOWN` — `device` (5 enum) + `keyCodes: number[]` (둘 다 필수)
- `ON_MEDIA_HIT` — `mediaHitTime: number` (필수, 초 단위)
- `ON_MEDIA_END` — 추가 필드 없음

**Trigger 입력 shape — backward-compat 유지:**
- 기존 string shortcuts `"ON_CLICK"`, `"ON_HOVER"`, `"ON_PRESS"`, `"AFTER_TIMEOUT"` + 기존 top-level `afterTimeoutSeconds` 그대로 작동
- 새 object form은 위 7종을 다 노출 (object form으로 기존 4종도 가능)

**`list_reactions` echo:**
- Directional transition은 raw passthrough (type, direction, matchLayers, duration, easing 그대로)
- Trigger 모든 새 타입도 raw passthrough — 각 타입별 필드 (delay/deprecatedVersion/device/keyCodes/mediaHitTime/timeout) 포함

**새 E2E 시나리오 17 (directional) + 18 (new triggers — 실용 subset)**

### v1.13 비-목표

- ON_KEY_DOWN 라이브 검증 (스키마/빌더만 — 데스크탑 환경 종속)
- ON_MEDIA_HIT / ON_MEDIA_END 라이브 검증 (비디오 노드 fixture 별도)
- "drag direction" 정보 (Figma 자체가 ON_DRAG에 방향 안 줌)
- preserveScrollPosition 노출 (v1+)
- Variables / conditional / set-variant action (v2 영역)

## 4. 사용자 API

### 4.1 TransitionInput 전체 union (확장)

```ts
type TransitionInput =
  // String shortcuts (기존)
  | "INSTANT" | "DISSOLVE" | "SMART_ANIMATE"
  // SimpleTransition 객체 (v0.10.0)
  | {
      type: "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE";
      duration?: number;
      easing?: EasingInput;
    }
  // DirectionalTransition 객체 (NEW)
  | {
      type: "MOVE_IN" | "MOVE_OUT" | "PUSH" | "SLIDE_IN" | "SLIDE_OUT";
      direction: "LEFT" | "RIGHT" | "TOP" | "BOTTOM";
      matchLayers?: boolean;       // default false
      duration?: number;            // default 0.3
      easing?: EasingInput;         // default "EASE_OUT"
    };
```

`direction`은 directional transition에서만 필수. `MOVE_IN/PUSH/SLIDE_IN` direction은 destination이 들어오는 방향, `MOVE_OUT/SLIDE_OUT` direction은 source가 떠나는 방향 — Figma 의미론 그대로.

### 4.2 TriggerInput 전체 union (확장)

```ts
type TriggerInput =
  // 기존 string shortcuts (backward compat)
  | "ON_CLICK" | "ON_HOVER" | "ON_PRESS" | "AFTER_TIMEOUT"
  // 새 object form — 추가 필드 없는 종류
  | { type: "ON_CLICK" | "ON_HOVER" | "ON_PRESS" | "ON_DRAG" | "ON_MEDIA_END" }
  // AFTER_TIMEOUT object form
  | { type: "AFTER_TIMEOUT"; timeout: number }
  // 마우스 click 계열
  | { type: "MOUSE_UP" | "MOUSE_DOWN"; delay?: number }
  // 마우스 hover 계열
  | { type: "MOUSE_ENTER" | "MOUSE_LEAVE"; delay?: number; deprecatedVersion?: boolean }
  // 키보드/컨트롤러
  | {
      type: "ON_KEY_DOWN";
      device: "KEYBOARD" | "XBOX_ONE" | "PS4" | "SWITCH_PRO" | "UNKNOWN_CONTROLLER";
      keyCodes: number[];  // 1+ 항목
    }
  // 미디어 (비디오)
  | { type: "ON_MEDIA_HIT"; mediaHitTime: number };
```

**Naming**: object form 안에서는 Figma-native 이름 사용 (`timeout`, `delay`, `mediaHitTime`). 기존 top-level `afterTimeoutSeconds`는 legacy로 string shortcut + `"AFTER_TIMEOUT"` 조합 한정으로 유지 (refine은 둘 중 하나만 검증).

### 4.3 사용 예시

```ts
// 1) MOVE_IN from right with matchLayers
{ targetFrameId, transition: { type: "MOVE_IN", direction: "RIGHT", matchLayers: true, duration: 0.4 } }
// 2) PUSH from bottom, default duration
{ targetFrameId, transition: { type: "PUSH", direction: "BOTTOM" } }
// 3) SLIDE_OUT to left with spring easing
{ targetFrameId, transition: { type: "SLIDE_OUT", direction: "LEFT", easing: "BOUNCY" } }
// 4) Drag trigger
{ trigger: { type: "ON_DRAG" }, ... }
// 5) Mouse up delayed
{ trigger: { type: "MOUSE_UP", delay: 0.2 }, ... }
// 6) Hover enter without delay
{ trigger: { type: "MOUSE_ENTER" }, ... }
// 7) Spacebar key
{ trigger: { type: "ON_KEY_DOWN", device: "KEYBOARD", keyCodes: [32] }, ... }
// 8) Media hit at 5s
{ trigger: { type: "ON_MEDIA_HIT", mediaHitTime: 5 }, ... }
// 9) AFTER_TIMEOUT object form (object 통일된 호출)
{ trigger: { type: "AFTER_TIMEOUT", timeout: 2 }, ... }
// 10) AFTER_TIMEOUT legacy form (backward compat, 변경 없음)
{ trigger: "AFTER_TIMEOUT", afterTimeoutSeconds: 2, ... }
```

### 5.3 ConnectionInput refine 업데이트

```ts
const ConnectionInput = z.object({
  sourceNodeId: z.string().min(1),
  trigger: TriggerInput.default("ON_CLICK"),
  afterTimeoutSeconds: z.number().positive().optional(),  // legacy only
  transition: TransitionInput.default("INSTANT"),
  action: ActionInput,
}).refine(
  // String "AFTER_TIMEOUT" needs top-level afterTimeoutSeconds.
  // Object trigger { type: "AFTER_TIMEOUT", timeout } self-contains.
  (v) => v.trigger !== "AFTER_TIMEOUT" || typeof v.afterTimeoutSeconds === "number",
  { message: "afterTimeoutSeconds is required when trigger is the string \"AFTER_TIMEOUT\"" }
);
```

기존 refine은 string 케이스만 검증. Object trigger는 z.object 안에서 자기 필드 검증으로 충분.

## 6. 빌더 함수 변화

### 6.1 buildTrigger 재작성

기존 시그니처 `buildTrigger(name: TriggerName, afterTimeoutSeconds?: number)` → 새 시그니처 `buildTrigger(input: TriggerInput, legacyAfterTimeoutSeconds?: number)`:

```ts
export function buildTrigger(
  input: TriggerInput,
  legacyAfterTimeoutSeconds?: number
): TriggerShape {
  // String shortcuts
  if (typeof input === "string") {
    if (input === "AFTER_TIMEOUT") {
      if (legacyAfterTimeoutSeconds === undefined) {
        throw new Error("buildTrigger: afterTimeoutSeconds required for string AFTER_TIMEOUT");
      }
      return { type: "AFTER_TIMEOUT", timeout: legacyAfterTimeoutSeconds };
    }
    return { type: input };
  }
  // Object form — emit Figma-native shape
  switch (input.type) {
    case "ON_CLICK":
    case "ON_HOVER":
    case "ON_PRESS":
    case "ON_DRAG":
    case "ON_MEDIA_END":
      return { type: input.type };
    case "AFTER_TIMEOUT":
      return { type: "AFTER_TIMEOUT", timeout: input.timeout };
    case "MOUSE_UP":
    case "MOUSE_DOWN":
      return { type: input.type, delay: input.delay ?? 0 };
    case "MOUSE_ENTER":
    case "MOUSE_LEAVE":
      return {
        type: input.type,
        delay: input.delay ?? 0,
        deprecatedVersion: input.deprecatedVersion ?? false,
      };
    case "ON_KEY_DOWN":
      return { type: "ON_KEY_DOWN", device: input.device, keyCodes: input.keyCodes };
    case "ON_MEDIA_HIT":
      return { type: "ON_MEDIA_HIT", mediaHitTime: input.mediaHitTime };
  }
}
```

`TriggerShape` 타입 확장:

```ts
export type TriggerShape =
  | { type: "ON_CLICK" | "ON_HOVER" | "ON_PRESS" | "ON_DRAG" | "ON_MEDIA_END" }
  | { type: "AFTER_TIMEOUT"; timeout: number }
  | { type: "MOUSE_UP" | "MOUSE_DOWN"; delay: number }
  | { type: "MOUSE_ENTER" | "MOUSE_LEAVE"; delay: number; deprecatedVersion: boolean }
  | { type: "ON_KEY_DOWN"; device: KeyboardDevice; keyCodes: number[] }
  | { type: "ON_MEDIA_HIT"; mediaHitTime: number };
```

### 6.2 buildTransition 확장

```ts
export function buildTransition(input: TransitionInput): TransitionShape {
  if (input === "INSTANT") return null;
  if (input === "DISSOLVE" || input === "SMART_ANIMATE") {
    return { type: input, duration: 0.3, easing: { type: "EASE_OUT" } };
  }
  // Object form — check directional vs simple by type
  if (
    input.type === "MOVE_IN" || input.type === "MOVE_OUT" ||
    input.type === "PUSH"    || input.type === "SLIDE_IN" ||
    input.type === "SLIDE_OUT"
  ) {
    return {
      type: input.type,
      direction: input.direction,
      matchLayers: input.matchLayers ?? false,
      duration: input.duration ?? 0.3,
      easing: resolveEasing(input.easing),
    };
  }
  // SimpleTransition 객체 (DISSOLVE / SMART_ANIMATE / SCROLL_ANIMATE)
  return {
    type: input.type,
    duration: input.duration ?? 0.3,
    easing: resolveEasing(input.easing),
  };
}
```

`TransitionShape` 확장:

```ts
export type TransitionShape =
  | null
  | { type: SimpleTransitionType; duration: number; easing: EasingShape }
  | {
      type: "MOVE_IN" | "MOVE_OUT" | "PUSH" | "SLIDE_IN" | "SLIDE_OUT";
      direction: "LEFT" | "RIGHT" | "TOP" | "BOTTOM";
      matchLayers: boolean;
      duration: number;
      easing: EasingShape;
    };
```

### 6.3 BuildInput 시그니처 업데이트

모든 reaction BuildInput에서 `trigger: TriggerName` → `trigger: TriggerInput`. `afterTimeoutSeconds?: number`는 legacy로 유지 (`buildTrigger`에 두 번째 인자로 전달).

```ts
export interface NavigateBuildInput {
  targetFrameId: string;
  trigger: TriggerInput;
  transition: TransitionInput;
  afterTimeoutSeconds?: number;  // legacy
}
// ... 동일 패턴으로 Scroll/Overlay/Close/Back/Url/SwapOverlay
```

각 builder 안의 `buildTrigger(input.trigger, input.afterTimeoutSeconds)` 호출은 그대로 유지.

## 7. Plugin handler 변화

`src/figma-plugin/code.ts`의 Command union 안 `trigger` 필드 타입을 새 TriggerInput union으로 확장, `transition` 필드도 directional 객체 variant 추가.

`handleListReactions`의 trigger/transition echo는 raw passthrough이므로 변경 없음 — Figma가 돌려주는 객체에 새 필드들 (`direction`, `matchLayers`, `delay`, `deprecatedVersion`, `device`, `keyCodes`, `mediaHitTime`) 포함된 채로 그대로 응답에 전달됨.

`src/mcp-server/types.ts`의 `ReactionConnectionInput`:
- `trigger` 타입을 `TriggerInputName | TriggerInputObject` union으로 widening
- `transition` 타입을 simple+directional 객체 포함하도록 widening

`ReactionSummary` (응답 측):
- `action.transition`에 directional fields (`direction`, `matchLayers`) 선택적 추가
- 트리거 응답 객체에 각 새 필드 선택적 노출

## 8. 새 E2E 시나리오

### 시나리오 17 — Directional transitions (5 sub-cases)
```
Setup: 기존 fixture (button A on screenA, screenB target) 재사용. 5번 navigate reactions 만들고 각각 transition 다르게.
(a) "RIGHT에서 MOVE_IN matchLayers true 0.4초"
   → transition: { type: "MOVE_IN", direction: "RIGHT", matchLayers: true, duration: 0.4 }
   Expected: list_reactions echoes type=MOVE_IN, direction=RIGHT, matchLayers=true, duration=0.4.
(b) "왼쪽에서 PUSH 0.3초"
   → { type: "PUSH", direction: "LEFT" }
   Expected: matchLayers=false default echoed.
(c) "BOTTOM에서 SLIDE_IN BOUNCY"
   → { type: "SLIDE_IN", direction: "BOTTOM", easing: "BOUNCY" }
   Expected: easing.type=BOUNCY echoed.
(d) "TOP으로 MOVE_OUT 0.5초"
   → { type: "MOVE_OUT", direction: "TOP", duration: 0.5 }
   Expected: direction=TOP, duration=0.5.
(e) "RIGHT로 SLIDE_OUT"
   → { type: "SLIDE_OUT", direction: "RIGHT" }
   Expected: type=SLIDE_OUT, direction=RIGHT.
```

각 sub-case 만들 때마다 `replaceExisting: true` 또는 clear 후 다시 — 한 버튼당 한 reaction 유지.

### 시나리오 18 — 새 Trigger (실용 subset; 3 sub-cases)
```
Setup: 기존 button + frame fixture. 마우스/드래그 트리거는 prototype play로 검증.
(a) ON_DRAG
   → trigger: { type: "ON_DRAG" }, action navigate
   Expected: list_reactions echoes trigger.type=ON_DRAG. Play 모드에서 드래그 시 네비게이트.
(b) MOUSE_UP delay 0.2
   → trigger: { type: "MOUSE_UP", delay: 0.2 }
   Expected: list_reactions echoes trigger.type=MOUSE_UP, delay=0.2. Play 모드 클릭 떼면 0.2초 후 네비게이트.
(c) MOUSE_ENTER → overlay
   → trigger: { type: "MOUSE_ENTER" }, action overlay (overlay frame 재사용)
   Expected: list_reactions echoes trigger.type=MOUSE_ENTER, delay=0, deprecatedVersion=false.
```

**검증 deferred (스키마/빌더만):**
- `ON_KEY_DOWN` — controller/key 동작 데스크탑 빌드 한정. 단위 테스트로 wire shape 정확성만 확보.
- `ON_MEDIA_HIT` / `ON_MEDIA_END` — 비디오 노드 fixture 별도. 단위 테스트로 wire shape만 검증.
