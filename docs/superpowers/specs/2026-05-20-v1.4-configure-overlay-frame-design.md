# figma-prototype-mcp v1.4 — `configure_overlay_frame` 설계 문서

- 작성일: 2026-05-20
- 상태: **🚫 BLOCKED — Figma 플랫폼 한계로 구현 불가**
- 기반 버전: v0.3.0 (v1 + v1.1 + v1.2 + v1.3 누적)

## ⚠️ 핵심 결론

**v1.4 구현은 시도했으나 라이브 API 검증 단계에서 차단됨**. Figma plugin API가 overlay 프레임의 세 속성을 모두 **read-only**로 노출하기 때문:

```ts
// @figma/plugin-typings/plugin-api.d.ts (line 7729-7737)
readonly overlayPositionType: OverlayPositionType
readonly overlayBackground: OverlayBackground
readonly overlayBackgroundInteraction: OverlayBackgroundInteraction
```

플러그인 코드에서 `frame.overlayPositionType = ...` 같은 할당을 시도하면 Figma가 `"no setter for property"` 에러를 반환한다. 검색해도 `setOverlay*Async` 같은 우회 setter 메서드가 typings에 존재하지 않는다. 즉 **이 세 속성은 Figma UI에서 디자이너가 수동 설정해야만 한다** — 플러그인 mutation API의 적용 범위 밖.

이 spec은 시도 기록 + 향후 Figma가 setter를 노출할 경우를 위한 설계 참고로 보존.

## 1. 원래 목적 (변경 없음)

v0.3.0의 `create_reactions`에서 overlay 액션을 만들 수 있지만, 모달이 **어디에 어떻게 보일지**는 사용자가 Figma UI에서 destination frame property로 직접 설정해야 한다. 자연어 프롬프트만으로 모달 패턴을 완결하려면 그 frame property를 MCP로 설정할 길이 필요하다 — 그 길을 만들려는 시도가 v1.4였다.

## 2. 시도된 설계 요약

새 MCP 툴 `configure_overlay_frame(frameId, position?, background?, closeOnClickOutside?)`를 추가, 핸들러가 destination frame의 세 property를 직접 설정. Partial-update 의미.

자세한 사용자-노출 API와 사용자 흐름은 §4–5 (아래 historical sections) 참조.

## 3. 발견된 한계의 디테일

라이브 시도 시 두 번 거부:

1. **`overlayBackgroundInteraction`, `overlayBackground` 할당** → `"no setter for property"`. 두 속성 모두 `readonly`.
2. **`overlayPositionType` 단독 할당** → 같은 에러. 마찬가지로 `readonly`.

또한 `@figma/plugin-typings`에서 `Overlay`로 grep해도 mutation 메서드 (`set*Async` / `update*` 등) 발견되지 않음.

추정: Figma의 prototype 데이터 모델 일부는 read-only로 노출되는 정책으로 보임. 디자이너가 직접 Figma UI에서 설정한 값만 시각적으로 반영되는 구조.

## 4. 향후 활성화 조건

Figma가 다음 중 하나라도 노출하면 본 spec을 재활성화 가능:

- `frame.overlayPositionType` 등의 `readonly` 제거
- `frame.setOverlayPositionTypeAsync(...)` 같은 새 setter 메서드 추가
- Document-레벨 prototype 설정 API
- 기타 overlay 프레임 property mutation 경로

활성화 시: 본 spec의 §5–10 (아래 historical 섹션)을 그대로 구현 plan으로 사용 가능. 핵심 매핑 로직(`mapPosition`/`mapBackground`/`mapInteraction`)은 이미 v1.4 시도 코드에 있었고 revert됐지만, git log에서 회수 가능 (`git show 0681aed`).

## 5. 사용자-노출 API (원래 설계 — 미구현)

```ts
configure_overlay_frame({
  frameId: string,
  position?:
    | "manual"
    | "center"
    | "top_left" | "top_center" | "top_right"
    | "bottom_left" | "bottom_center" | "bottom_right",
  background?: "none" | "dim",
  closeOnClickOutside?: boolean
})
```

응답: `{ frameId, status, error?, applied: { ...적용된 필드만 } }`.

Partial update — 미지정 필드는 기존 값 유지.

## 6. 매핑 (원래 설계 — 미구현)

| 사용자 입력 | Figma frame property | Figma 값 |
|---|---|---|
| `position: "center"` 등 | `frame.overlayPositionType` | `"CENTER"` 등 (8개 SHOUTING_SNAKE_CASE) |
| `background: "none"` | `frame.overlayBackground` | `{ type: "NONE" }` |
| `background: "dim"` | `frame.overlayBackground` | `{ type: "SOLID_COLOR", color: { r: 0, g: 0, b: 0, a: 0.5 } }` |
| `closeOnClickOutside: true/false` | `frame.overlayBackgroundInteraction` | `"CLOSE_ON_CLICK_OUTSIDE"` / `"NONE"` |

## 7. 사용자가 지금 할 일 (대안)

Overlay 프레임의 시각 설정은 **Figma UI에서 디자이너가 직접 설정** 필요:

1. Figma에서 destination frame 선택
2. 우측 패널 Prototype 탭
3. "Show as overlay" 켜진 상태에서:
   - **Position**: Manual / Center / Top Left / ... 중 선택
   - **Background**: None / 색상 선택 (alpha 포함)
   - **Close when clicking outside**: 체크박스

MCP는 `create_reactions`로 wiring만 자동화. 시각 설정은 디자이너 작업으로 남음.

## 8. 학습 — Figma plugin API mutation 범위 확인 패턴

v1.4 시도가 v1.1/v1.3 라이브-API-surprise보다 더 깊은 교훈: **유사 시나리오에서 setter 존재 여부를 spec 단계에서 확인하라**.

체크 순서:
1. `@figma/plugin-typings/plugin-api.d.ts`에서 해당 property 정의 찾기
2. `readonly` 키워드 검사 — 있으면 직접 할당 불가
3. 없으면 `set*Async` / `update*` 같은 메서드 패턴 검색
4. 두 검사 모두 통과하면 라이브 검증 (기존 패턴 유지)

이 체크가 spec 단계에 있었다면 v1.4 5개 구현 commit + 1차 라이브 시도 사이클이 모두 절약됐을 것. 다음 spec부터 의무 단계로 추가.
