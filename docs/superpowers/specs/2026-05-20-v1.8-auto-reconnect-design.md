# figma-prototype-mcp v1.8 — Plugin auto-reconnect 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.6.0
- 영향 범위: `src/figma-plugin/ui.html`만. sandbox / MCP server / relay / 빌더 무관.

## 1. 목적

v0.6.0의 plugin은 reload 시 자동 connect는 되지만, **세션 중 WebSocket이 끊기면** (relay 재시작, 네트워크 blip 등) 사용자가 Connect 버튼을 다시 눌러야 한다. v1.8은 의도하지 않은 disconnect를 자동 감지하고 backoff로 재연결한다.

## 2. spec-단계 readonly/setter 체크

순수 UI/타이머 로직만 추가 — Figma plugin API 호출 없음. setReactionsAsync 같은 mutation 경로 없음. ✅ 차단 없음.

## 3. 스코프

### v1.8 포함
- `ws.onclose`에서 의도하지 않은 닫힘 감지 → 지수 backoff retry 루프
- Backoff: 1s → 2s → 4s → 8s → 16s → 30s (cap), 최대 횟수 없음
- 성공 시 backoff 1s로 리셋
- 사용자 Disconnect 클릭 → 즉시 루프 중단 (timer clear + flag set)
- Retry 진행 중 UI: input은 disabled 유지, Disconnect 버튼 계속 보임. status가 "Reconnecting to {channel} in {N}s…"

### v1.8 비-목표
- 최대 retry 횟수 제한
- 카운트다운 tick (현재 단계는 backoff 시간을 한 번에 status에 표시; 1s마다 갱신은 안 함)
- 수동 "Retry now" 버튼
- 네트워크 상태 detection beyond WebSocket close

## 4. 상태 머신

```
[connected]
  ↓ ws.onclose (userDisconnected = false)
[reconnecting, attempt 0, backoff 1s]
  ↓ wait 1s
[reconnecting, attempt 1] — ws connecting…
  ├─ success → [connected], backoff reset to 1s
  └─ fail (close) → [reconnecting, attempt 2, backoff 2s]
  ↓ wait 2s
[reconnecting, attempt 2] — ws connecting…
  ├─ success → [connected]
  └─ fail → [reconnecting, attempt 3, backoff 4s]
  ↓ wait 4s
...
[reconnecting, attempt N, backoff 30s]   ← cap
  ↓ wait 30s
...

At any point in [reconnecting]:
  user clicks Disconnect → clear timer, userDisconnected = true → [disconnected]
```

## 5. UI 동작 디테일

- **Retry 시작**: `setStatus('Reconnecting to ' + channel + ' in ' + (delay/1000) + 's…', 'pending')`
- **Attempt 시작** (timer 만료 직후, ws.connect 직전): `setStatus('Reconnecting to ' + channel + '…', 'pending')`
- **Attempt 성공** (join ACK 수신): 기존 `setConnectedUI`, 추가로 `retryDelay = 1000`
- **Attempt 실패** (ws.onclose 다시): 다음 backoff 계산해서 status 갱신 + 새 timer 설정
- **Disconnect 클릭**: 기존 핸들러 + 추가 `userDisconnected = true`, `if (retryTimer) clearTimeout(retryTimer)`

Connected 상태 진입(`setConnectedUI`) 시 `userDisconnected = false`로 초기화 (다음 끊김에 다시 retry 시작하도록).

## 6. 코드 변경

`src/figma-plugin/ui.html` `<script>` 블록에 다음 추가:

```js
let userDisconnected = false;
let retryTimer = null;
let retryDelay = 1000;
const MAX_RETRY_DELAY = 30000;
```

`connect()` 시작에 retry 상태 정리:

```js
function connect() {
  if (retryTimer) { clearTimeout(retryTimer); retryTimer = null; }
  userDisconnected = false;
  // ... 기존 로직
}
```

`ws.onclose` 핸들러 교체:

```js
ws.onclose = () => {
  if (userDisconnected) {
    setStatus('Disconnected', 'err');
    setDisconnectedUI();
    return;
  }
  // Unintended close — schedule a retry with backoff.
  const channel = channelInput.value.trim();
  setStatus('Reconnecting to ' + channel + ' in ' + (retryDelay / 1000) + 's…', 'pending');
  retryTimer = setTimeout(() => {
    retryTimer = null;
    setStatus('Reconnecting to ' + channel + '…', 'pending');
    retryDelay = Math.min(retryDelay * 2, MAX_RETRY_DELAY);
    connect();
  }, retryDelay);
};
```

`setConnectedUI` 끝에:

```js
function setConnectedUI(channel) {
  // ... 기존
  retryDelay = 1000;   // reset
}
```

`disconnectBtn.onclick` 교체:

```js
disconnectBtn.onclick = () => {
  userDisconnected = true;
  if (retryTimer) { clearTimeout(retryTimer); retryTimer = null; }
  if (ws) ws.close();
  setStatus('Disconnected', 'err');
  setDisconnectedUI();
};
```
