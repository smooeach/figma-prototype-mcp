# figma-prototype-mcp v1.6 — URL 액션 옵션 + inspection 보강 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.4.0
- 직전 spec: v1.5 (`docs/superpowers/specs/2026-05-20-v1.5-back-url-swap-design.md`)

## 1. 목적

v0.4.0의 `list_reactions`는 URL 액션을 `{ type: "URL" }`로만 노출. URL 문자열이 응답에 없어서 시나리오 3 inspection이 "이 버튼은 URL로 가요" 수준에 그침. `create_reactions`도 `openInNewTab` 옵션을 노출하지 않아 항상 같은 탭 동작.

v1.6는 URL 액션 관련 두 폴리시를 한 묶음으로:
1. `create_reactions` URL variant에 `openInNewTab?: boolean` 옵셔널 입력
2. `list_reactions` 응답에 `url`, `openInNewTab` 노출

## 2. spec-단계 readonly/setter 체크

Figma `@figma/plugin-typings`에서:

```ts
| { readonly type: 'URL'; url: string; openInNewTab?: boolean }
```

- `url`, `openInNewTab` 모두 first-class Action union 필드.
- Mutation 경로는 기존 `setReactionsAsync` — 새 setter 호출 없음.
- ✅ 차단 없음.

## 3. 스코프

### v1.6 포함
- `create_reactions` URL 액션 입력에 `openInNewTab?: boolean` (기본 false).
- `buildUrlReaction`이 `openInNewTab` 받아 wire shape에 emit.
- `handleListReactions`가 응답 `action` 객체에 `url`, `openInNewTab` 둘 다 노출.
- `ReactionSummary` 타입 보강.

### v1.6 비-목표
- 다른 액션 타입 (Set Variable / Conditional / Media-runtime).
- Transition `duration` / `easing` 사용자 커스터마이즈 (별도 polish 후보).
- AFTER_TIMEOUT 등 새 trigger.

## 4. API 변화

### 4.1 입력 (`create_reactions`)

```ts
action: { type: "url", url: string, openInNewTab?: boolean }
```

`openInNewTab` 미지정 = `false` (현재 v0.4.0 강제값).

### 4.2 출력 (`list_reactions`)

URL 액션을 가진 노드의 응답:

```ts
{
  index, trigger,
  action: {
    type: "URL",
    url: string,                 // ← NEW
    openInNewTab: boolean,       // ← NEW
    destinationId: undefined,    // URL은 destination 없음
    destinationName: undefined,
    transition: undefined        // URL에 transition 의미 없음
  }
}
```

| `src/figma-plugin/reaction-builder.ts` | `UrlBuildInput`에 `openInNewTab?: boolean`. `BuiltAction.URL`에 `openInNewTab: boolean`. `buildUrlReaction`이 `input.openInNewTab ?? false`를 emit. |
| `src/figma-plugin/code.ts` | Command union URL variant에 `openInNewTab?: boolean`. handleCreateReactions의 url 분기에서 buildUrlReaction에 `openInNewTab` 전달. handleListReactions output에 `url`, `openInNewTab` 추가. |
