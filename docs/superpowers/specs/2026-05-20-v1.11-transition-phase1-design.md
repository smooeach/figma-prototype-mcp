# figma-prototype-mcp v1.11 — Transition Phase 1 (custom duration + 7 easings) 설계 문서

- 작성일: 2026-05-20
- 상태: 초안 (사용자 리뷰 대기)
- 기반 버전: v0.9.0
- 시리즈: 4-phase 트랜지션 노출 중 **Phase 1**
- 참조: `docs/natural-language-mapping-dictionary-v2.3.md` (사용자 제공 v2.3)

## 1. 목적

v0.9.0의 transition은 3개 단축어(`"INSTANT"` / `"DISSOLVE"` / `"SMART_ANIMATE"`)만 노출. duration·easing이 0.3s + EASE_OUT로 고정이라 디자이너가 "느린 페이드", "강조 부드럽게", "기대감 후 등장" 같은 자연어를 표현 못 함.

Phase 1은 **simple transition 3종에 대해 duration + easing을 자유롭게 지정 가능**하게 함. Directional·Spring·Custom은 후속 phase에서.

## 2. spec-단계 readonly/setter 체크

Transition·Easing 객체는 setReactionsAsync 인자로 wire에 직접 넣는 객체 (read-only 마커는 read shape용). v1.3 OVERLAY action 객체와 동일 패턴. ✅ 차단 없음.

## 3. 스코프

### v1.11 포함
- Transition shape을 단축어 또는 nested object 둘 다 허용:
  - 단축어: `"INSTANT"` (null로 emit), `"DISSOLVE"`, `"SMART_ANIMATE"` (defaults 0.3s + EASE_OUT)
  - Nested: `{ type: "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE", duration?, easing? }`
- Easing 7종:
  - `"LINEAR"` / `"EASE_IN"` / `"EASE_OUT"` / `"EASE_IN_AND_OUT"`
  - `"EASE_IN_BACK"` / `"EASE_OUT_BACK"` / `"EASE_IN_AND_OUT_BACK"`
- Duration: `z.number().positive().max(10)` (초 단위)
- Defaults (nested에서 미지정): duration=0.3, easing="EASE_OUT"
- `list_reactions` 응답 transition에 `duration` 및 `easing` 노출

### v1.11 비-목표
- Directional transitions: MOVE_IN / MOVE_OUT / PUSH / SLIDE_IN / SLIDE_OUT → **Phase 3 (v1.13)**
- Spring easings (GENTLE/QUICK/BOUNCY/SLOW) → **Phase 2 (v1.12)**
- Custom cubic-bezier / custom spring → **Phase 4 (v1.14)**
- matchLayers → Phase 3과 함께
- `SCROLL_ANIMATE` 라이브 검증 (이번엔 스키마에 추가만 — Phase 3에서 함께 테스트)

## 4. 사용자 API

### 4.1 ConnectionInput.transition 입력

```ts
transition?:
  | "INSTANT" | "DISSOLVE" | "SMART_ANIMATE"
  | {
      type: "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE",
      duration?: number,             // 초, default 0.3, 범위 0.01–10
      easing?: EasingInputName        // default "EASE_OUT"
    }

type EasingInputName =
  | "LINEAR" | "EASE_IN" | "EASE_OUT" | "EASE_IN_AND_OUT"
  | "EASE_IN_BACK" | "EASE_OUT_BACK" | "EASE_IN_AND_OUT_BACK";
```

### 4.2 단축어 매핑

| 입력 | wire shape |
|---|---|
| `"INSTANT"` | `null` (no transition object) |
| `"DISSOLVE"` | `{ type: "DISSOLVE", duration: 0.3, easing: { type: "EASE_OUT" } }` |
| `"SMART_ANIMATE"` | `{ type: "SMART_ANIMATE", duration: 0.3, easing: { type: "EASE_OUT" } }` |

### 4.3 Nested object 매핑 예시

```ts
// 입력
transition: { type: "DISSOLVE", duration: 0.5, easing: "EASE_OUT_BACK" }

// wire shape
transition: {
  type: "DISSOLVE",
  duration: 0.5,
  easing: { type: "EASE_OUT_BACK" }
}
```

### 4.4 `list_reactions` 응답

기존 `transition: { type: string, duration?: number }` 에서 확장:

```ts
transition?: {
  type: string,
  duration?: number,
  easing?: { type: string }      // ← NEW
}
```

## 5. Zod 스키마

```ts
const EasingInputEnum = z.enum([
  "LINEAR", "EASE_IN", "EASE_OUT", "EASE_IN_AND_OUT",
  "EASE_IN_BACK", "EASE_OUT_BACK", "EASE_IN_AND_OUT_BACK",
]);

const SimpleTransitionObject = z.object({
  type: z.enum(["DISSOLVE", "SMART_ANIMATE", "SCROLL_ANIMATE"]),
  duration: z.number().positive().max(10).optional(),
  easing: EasingInputEnum.optional(),
});

const TransitionInput = z.union([
  z.literal("INSTANT"),
  z.literal("DISSOLVE"),
  z.literal("SMART_ANIMATE"),
  SimpleTransitionObject,
]);

ConnectionInput.transition = TransitionInput.default("INSTANT");
```

## 6. 빌더 함수 변화

### 6.1 `buildTransition` 시그니처 확장

기존:
```ts
function buildTransition(name: TransitionName): TransitionShape
```

새:
```ts
type TransitionInputResolved =
  | { type: "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE", duration: number, easing: EasingName }
  | null;

function buildTransition(input: TransitionInputUnion): TransitionShape {
  // 1) "INSTANT" or undefined → null
  // 2) string shortcut → expand to object with defaults
  // 3) nested object → resolve defaults for missing duration/easing
  // 4) emit { type, duration, easing: { type: easing } }
}
```

### 6.2 모든 7개 builder

`buildNavigateReaction` / `buildScrollReaction` / `buildOverlayReaction` / `buildSwapOverlayReaction`은 transition을 받음 — 4개 builder의 input interface가 `TransitionInputUnion`을 받도록 확장.

`buildCloseReaction` / `buildBackReaction` / `buildUrlReaction`은 transition 무관 — 그대로.

### 6.3 BuiltReaction 타입

`TransitionShape` 확장:
```ts
type EasingName =
  | "LINEAR" | "EASE_IN" | "EASE_OUT" | "EASE_IN_AND_OUT"
  | "EASE_IN_BACK" | "EASE_OUT_BACK" | "EASE_IN_AND_OUT_BACK";

type TransitionShape =
  | null
  | { type: "DISSOLVE" | "SMART_ANIMATE" | "SCROLL_ANIMATE", duration: number, easing: { type: EasingName } };
```

(Directional 변종은 Phase 3에서 union에 추가.)

## 7. Plugin handler 변화

`handleCreateReactions`: connection.transition을 builder들에 그대로 전달 (이미 그렇게 하고 있음). 변경 없음.

`handleListReactions`: 현재 `transition: action.transition`만 echo. transition이 객체면 `type` + `duration` + `easing` 통째로 echo 되도록 type 확장만.

## 11. 라이브 리스크

- Figma가 받는 transition 객체에 모든 필드를 정확히 요구하는지 확인 — `easing: { type: "EASE_OUT" }` 같은 nested 객체. typings에 따르면 정상.
- `SCROLL_ANIMATE`는 scroll-driven 동작이라 일반 navigate에선 의미 모호. Phase 1에선 스키마만 추가, 라이브 검증은 Phase 3 SCROLL_TO action과 함께.
- `EASE_IN_AND_OUT_BACK` 등 덜 알려진 easing이 typings에 있다고 정상 동작 보장 X — 라이브 검증 1회 필요.
